NMPScript 使用文档

版本 1.0 | 最后更新: 2026年7月4日

简介

NMPScript 是一个用 Shell 脚本编写的 Linux 系统自动化安装工具，可以在 Linux 系统上快速部署 LNMP（Nginx + MariaDB + PHP）、LNMPA（Nginx + Apache + MariaDB + PHP）、LAMP（Apache + MariaDB + PHP）生产环境。

主要特点：

  • 一键安装，无需手动配置
  • 支持多种 PHP 版本（8.1、8.2、8.3、8.4）
  • 支持 MariaDB 10.5、10.6、10.11 版本
  • 支持二进制安装（快速）和源码编译安装
  • 包含常用 PHP 扩展和缓存组件
  • 提供完整的虚拟主机管理功能

安装指南

下载安装包

克隆项目
wget https://github.com/laingyulee/nmpscript/archive/refs/heads/main.tar.gz tar -zxvf main.tar.gz cd nmpscript-main
配置安装参数（可选）

编辑 lnmp.conf 文件自定义安装参数：

vim lnmp.conf

主要配置项：

  • Nginx_Modules_Options - Nginx 附加模块
  • PHP_Modules_Options - PHP 附加模块
  • MariaDB_Data_Directory - 数据库数据目录
  • Default_Website_Directory - 默认网站目录
  • Enable_Swap - 是否启用 Swap（内存不足时推荐开启）
运行安装脚本
chmod +x install.sh ./install.sh lnmp

安装模式选择

命令 说明 安装内容
./install.sh lnmp LNMP 模式 Nginx + MariaDB + PHP
./install.sh lamp LAMP 模式 Apache + MariaDB + PHP
./install.sh lnmpa LNMPA 模式 Nginx + Apache + MariaDB + PHP
./install.sh db 仅安装数据库 MariaDB
./install.sh nginx 仅安装 Nginx Nginx
./install.sh mphp 安装多版本 PHP PHP（需要先安装 Nginx）

使用 screen 防止编译断线

源码编译安装可能需要 1-3 小时，期间如果 SSH 连接中断会导致安装失败。建议使用 screen 创建虚拟终端来保持会话。

重要
使用二进制安装时通常不需要 screen（仅需 10-30 分钟），但建议在不稳定的网络环境下始终使用。

安装 screen：

# CentOS / Rocky Linux / AlmaLinux yum install screen -y # Debian / Ubuntu apt-get install screen -y

创建虚拟终端会话：

# 创建名为 lnmp 的会话 screen -S lnmp

在会话中执行安装脚本：

chmod +x install.sh ./install.sh lnmp

常用 screen 命令：

  • Ctrl + A 然后按 D - 分离会话（保持后台运行）
  • screen -ls - 查看所有会话列表
  • screen -r lnmp - 重新连接到名为 lnmp 的会话
  • exit - 在会话内执行，关闭当前会话
提示
如果 SSH 连接中断，只需重新登录服务器，然后执行 screen -r lnmp 即可恢复安装进程。

安装过程

安装过程中会提示选择以下内容：

  1. 数据库版本：MariaDB 10.5 / 10.6 / 10.11
  2. 安装方式：二进制安装（推荐）或源码编译
  3. PHP 版本：PHP 8.1 / 8.2 / 8.3 / 8.4
  4. 是否启用 InnoDB：MariaDB 存储引擎
  5. PHP 扩展：选择需要的 PHP 扩展
提示
直接按回车将使用默认推荐配置：MariaDB 10.11（二进制安装）+ PHP 8.4。

使用方法

服务管理

命令 说明
lnmp start 启动所有服务
lnmp stop 停止所有服务
lnmp restart 重启所有服务
lnmp status 查看服务状态
lnmp killall 强制终止所有服务

虚拟主机管理

命令 说明
lnmp vhost add 添加虚拟主机
lnmp vhost list 列出所有虚拟主机
lnmp vhost del 删除虚拟主机
lnmp vhost edit 编辑虚拟主机配置

添加虚拟主机时会提示输入：

  • 域名（如 example.com）
  • 网站根目录
  • 是否启用日志
  • 是否启用 SSL

数据库管理

命令 说明
lnmp database add 添加数据库
lnmp database list 列出所有数据库
lnmp database del 删除数据库
lnmp database edit 修改数据库用户密码

FTP 管理

命令 说明
lnmp ftp add 添加 FTP 用户
lnmp ftp list 列出所有 FTP 用户
lnmp ftp del 删除 FTP 用户

SSL 证书

命令 说明
lnmp ssl add 添加 SSL 证书
lnmp ssl del 删除 SSL 证书

支持两种 SSL 证书方式：

  • 自签名证书：快速生成，仅用于测试
  • Let's Encrypt：免费自动续期的正式证书

常见问题

安装失败：磁盘空间不足

源码编译安装需要约 5GB 空间，如果空间不足会编译失败。

解决方案：

  • 使用二进制安装（仅需约 500MB）
  • 清理系统缓存：yum clean all 或 apt clean
  • 删除旧编译文件：rm -rf /tmp/mariadb-* /tmp/php-*

ARM 架构安装问题

MariaDB 官方不提供 ARM64 (aarch64) 架构的二进制包，ARM 服务器必须使用源码编译安装。

建议：

  • 确保内存至少 2GB
  • 确保磁盘空间至少 10GB
  • 编译时间约 1-2 小时

lnmp 命令不存在

如果使用 ./install.sh nginx 或 ./install.sh db 单独安装数据库，不会创建 lnmp 命令。

解决方案：

  • 使用完整安装：./install.sh lnmp
  • 或手动创建：cp /root/nmpscript-main/conf/lnmp /bin/lnmp && chmod +x /bin/lnmp

数据库连接失败

检查 MariaDB 服务状态：

systemctl status mariadb # 或 service mariadb status

常见原因：

  • 服务未启动：lnmp start
  • 密码错误：使用 lnmp database edit 重置密码
  • 端口未开放：检查防火墙设置

PHP 页面空白或 500 错误

检查 PHP-FPM 服务：

systemctl status php-fpm # 查看错误日志 tail -f /usr/local/php/var/log/php-fpm.log

常见原因：

  • PHP 扩展缺失：检查需要的扩展是否安装
  • 权限问题：检查网站目录权限
  • 配置错误：检查 php.ini 配置

重置 MariaDB root 密码

使用工具脚本重置密码：

./tools/reset_mysql_root_password.sh

升级与卸载

升级软件版本

使用升级脚本：

./upgrade.sh

可选升级：

  • Nginx 版本
  • MariaDB 版本
  • PHP 版本
升级前注意
升级前请备份数据库和重要配置文件。MariaDB 升级可能需要数据库迁移。

卸载 LNMP

使用卸载脚本：

./uninstall.sh

卸载过程会：

  • 停止所有服务
  • 备份数据库数据（可选）
  • 删除安装的程序文件
  • 保留网站数据和数据库备份
